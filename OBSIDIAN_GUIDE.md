# Obsidian 使用指南

## 第一步：下载安装Obsidian

1. 访问 https://obsidian.md/
2. 点击 "Download for Windows"
3. 运行安装程序，按默认设置安装

## 第二步：打开知识库

### 方法1: 直接打开文件夹

1. 打开Obsidian
2. 点击左下角的 "打开另一个库" 按钮（文件夹图标）
3. 选择 "打开文件夹作为库"
4. 浏览到 `D:\code\Knowledge_DB\knowledge_base`
5. 点击 "选择文件夹"

### 方法2: 创建新库并添加文件夹

1. 打开Obsidian
2. 点击左下角的 "打开另一个库" 按钮
3. 选择 "创建新库"
4. 库名称: `产品经理知识库`
5. 文件夹位置: `D:\code\Knowledge_DB`
6. 点击 "创建"

## 第三步: 配置Obsidian（推荐）

### 设置中文界面

1. 点击左下角齿轮图标 ⚙️ → "设置"
2. 选择 "外观"
3. 语言: 选择 "中文"

### 配置Markdown显示

1. 设置 → "编辑器"
2. 开启 "严格换行"
3. 开启 "可读行长度"
4. 开启 "显示行号"

### 配置文件管理

1. 设置 → "文件与链接"
2. 开启 "检测所有类型的链接"
3. 开启 "自动更新内部链接"

## 第四步: 浏览知识库

### 文件树视图

1. 点击左侧边栏的 "文件" 图标（或按 `Ctrl+E`）
2. 你会看到以下文件夹结构：
   ```
   knowledge_base/
   ├── product-design/      # 产品设计
   ├── user-research/       # 用户研究
   ├── data-analysis/       # 数据分析
   ├── project-management/  # 项目管理
   ├── strategy/            # 产品策略
   └── industry-insights/   # 行业洞察
   ```

### 查看文章

1. 点击任意文件夹展开
2. 点击日期文件夹（如 `2026-05-26`）
3. 点击 `.md` 文件查看内容
4. 文章会在右侧主区域显示

### 文章格式

每个文章包含：
```markdown
# 文章标题

**作者**: xxx
**发布时间**: xxx
**原文链接**: [url](url)
**标签**: xxx

---

## 核心观点（200字）
...

## 方法论/框架（300字）
...

## 关键案例（200字）
...

## 实践建议（200字）
...

## 关键词标签
...
```

## 第五步: 搜索功能

### 全局搜索

1. 按 `Ctrl+Shift+F` 打开搜索
2. 输入关键词
3. 搜索结果会高亮显示

### 搜索技巧

- 搜索标题: `file:文章标题`
- 搜索标签: `tag:产品设计`
- 搜索内容: `核心观点`
- 正则表达式: `/正则/`

### 过滤搜索

- 按文件夹: `path:product-design`
- 按日期: `path:2026-05-26`
- 按标签: `tag:#产品设计`

## 第六步: 使用标签

### 查看所有标签

1. 点击左侧边栏的 "标签" 图标
2. 查看所有标签列表
3. 点击标签查看相关文章

### 常用标签

- `#产品设计`
- `#用户研究`
- `#数据分析`
- `#项目管理`
- `#产品策略`
- `#行业洞察`

### 添加标签

在文章中添加标签：
```markdown
#标签名称
```

## 第七步: 建立链接

### 双向链接

在文章中创建链接：
```markdown
[[文章标题]]
[[文章标题|显示文本]]
```

### 链接到特定位置

```markdown
[[文章标题#标题]]
```

### 查看反向链接

1. 打开任意文章
2. 点击右上角的 "更多" 按钮
3. 选择 "反向链接" 或 "出链"

## 第八步: 知识图谱

### 查看图谱

1. 点击左侧边栏的 "图谱" 图标
2. 查看所有文章的链接关系
3. 点击节点跳转到文章

### 本地图谱

1. 打开任意文章
2. 点击右上角的 "更多" 按钮
3. 选择 "本地图谱"
4. 查看当前文章的链接关系

## 第九步: 插件推荐

### 安装插件

1. 设置 → "第三方插件"
2. 点击 "浏览"
3. 搜索并安装插件

### 推荐插件

1. **Tag Wrangler** - 标签管理
   - 批量重命名标签
   - 合并标签

2. **Dataview** - 数据查询
   - 查询文章列表
   - 统计文章数量

3. **Calendar** - 日历视图
   - 按日期查看文章
   - 快速导航

4. **Advanced Tables** - 表格增强
   - 更好的表格编辑
   - 表格公式

5. **Kanban** - 看板视图
   - 任务管理
   - 项目跟踪

## 第十步: 同步设置

### 使用Git同步

知识库已配置Git，可以：
```bash
# 拉取最新内容
git pull

# 推送更改
git add .
git commit -m "更新知识库"
git push
```

### 使用Obsidian Sync（付费）

1. 设置 → "第三方插件"
2. 启用 "Obsidian Sync"
3. 登录Obsidian账号
4. 同步知识库

### 使用其他云盘

将知识库文件夹放在：
- OneDrive
- Google Drive
- Dropbox
- 坚果云

## 工作流建议

### 日常学习流程

1. **早上**: 打开Obsidian，查看新文章
2. **学习**: 阅读文章，添加笔记
3. **整理**: 建立链接，添加标签
4. **回顾**: 查看图谱，复习知识

### 知识管理流程

1. **收集**: 运行爬虫获取新文章
2. **整理**: AI自动总结生成
3. **连接**: 建立文章间的链接
4. **应用**: 搜索和使用知识

### 团队协作流程

1. **同步**: Git推送和拉取
2. **讨论**: 使用GitHub Issues
3. **贡献**: 提交Pull Request
4. **分享**: 分享仓库链接

## 常见问题

### Q: 打开后没有看到文件？
A: 检查是否选择了正确的文件夹 `knowledge_base`

### Q: 中文显示乱码？
A: 设置 → 外观 → 语言 → 中文

### Q: 如何批量重命名文件？
A: 使用插件 "Batch Rename" 或命令行

### Q: 如何导出为PDF？
A: 右键文件 → "导出为PDF"

### Q: 如何备份？
A: 使用Git推送到GitHub

### Q: 如何同步到手机？
A: 使用Obsidian移动版 + 云盘同步
